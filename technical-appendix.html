1. Introduction and purpose of the STEPS tool

STEPS is a scenario based decision aid to support planning and scale up of Field Epidemiology Training Programmes (FETP) in India. The tool converts programme design choices and cost inputs into coherent economic outputs that can be explained, compared, and stress tested.

The tool has three purposes. First, it provides transparent cost totals for alternative programme configurations and scale up pathways. Second, it translates stated preference evidence into welfare measures (willingness to pay) to reflect stakeholder value for programme design features. Third, it constructs epidemiological economic benefits as avoided outbreak losses using literature anchored monetary values, enabling benefit cost and net benefit comparisons under sensitivity scenarios.

2. Overview of the preference study and mixed logit model

The preference component is based on a discrete choice experiment that elicits stakeholder trade offs over FETP design attributes. A mixed logit model is used to allow preference heterogeneity and to support welfare calculations (willingness to pay) for complete programme packages.

Un,j = Vn,j + εn,j
Vn,j = ASC·1{j is programme} + β'Xj + βc·Costj

Explanation: Each respondent n assigns utility U to alternative j. Systematic utility V depends on a programme constant (relative to opting out), a vector of attribute indicators X, and the cost attribute. The error term ε captures unobserved factors and is assumed independent and identically distributed extreme value for logit based choice probabilities.

Pn,j = ∫ [ exp(Vn,j(β)) / Σk exp(Vn,k(β)) ] f(β | θ) dβ

Explanation: Mixed logit integrates the standard logit probability over a distribution of random coefficients β with parameters θ. The tool uses simulated integration (a finite number of draws) to approximate the integral.

Implementation note: The tool stores the estimated model coefficients (and any random coefficient settings) in the script that runs in the browser. Scenario specific WTP outputs are computed from these embedded estimates.

3. Program design attributes and fixed response capacity logic

Scenario inputs correspond to the attributes used in the preference study. Each scenario selects one level for each attribute, and the tool calculates outputs for the complete package.

Fixed response capacity explanation (within 7 days)

Response capacity is fixed at within 7 days because it is treated as a system performance outcome that the training and supporting structures are intended to deliver. Allowing users to set response time as an input would mix inputs and outputs and would risk double counting benefits if outbreak savings were also constructed from improved response. Fixing response capacity ensures a clean separation: users choose training inputs and scale, and the tool reports economic value including outbreak savings without users directly choosing the outbreak performance outcome.

This design also supports interpretability. With response time fixed, differences across scenarios are driven by programme design (type, mentoring, delivery mode, incentive) and cost, rather than by a user chosen performance outcome.

4. Cost construction methodology

STEPS uses a single cost input from the user, cost per trainee per month, and scales it by duration, cohort size, and number of cohorts. The tool then optionally decomposes and reports cost components using cost shares derived from WHO, NIE, and NCDC costing data provided below. This decomposition is for transparency; the primary cost driver in the simulation is the user selected cost per trainee per month.

Cost per trainee (full programme) = Cost per trainee per month × Duration (months)

Explanation: The user selects a monthly cost level and a programme duration. The tool converts these to a full programme cost per trainee.

Financial programme cost (per cohort) = Cost per trainee (full programme) × Trainees per cohort

Explanation: Cohort cost scales linearly with the number of trainees trained in the cohort.

Total financial programme cost (scenario) = Financial programme cost (per cohort) × Number of cohorts

Explanation: Scale up is represented by the number of cohorts delivered under the selected programme configuration.

5. Cost component shares and scaling logic

Cost share profiles are constructed from the provided costing table (Section 5.3). For each programme type, STEPS computes the share of grand total costs accounted for by major categories and uses these shares to present a transparent breakdown consistent with observed cost structures.

Shareg = Cost componentg / Grand total cost

Explanation: Each category g (for example travel expenses) is expressed as a proportion of the grand total cost for that programme type in the costing dataset.

Implied component cost (scenario) = Total cost (scenario) × Shareg

Explanation: Once a scenario total cost is computed from user inputs, the tool multiplies the total by the category shares to report a consistent breakdown.

Why the cost slider remains within the preference study range

WTP and uptake calculations are identified from the cost levels included in the preference study. Using costs far outside that range can require extrapolation, which can reduce credibility because the underlying model has limited evidence to support behaviour far beyond observed attribute variation. STEPS therefore keeps the cost input within the study range and provides sensitivity tools to explore nearby alternatives.

6. Program cost per trainee and cohort calculations

For each scenario, the tool reports at minimum: cost per trainee for the full programme, cost per cohort, and total cost at the chosen scale. When a cost component breakdown is displayed, it is derived by applying the shares in Section 5 to the scenario total cost.

Cost per cohort = (Cost per month × Duration months) × Cohort size

Explanation: This converts the monthly cost input into a cohort total using duration and cohort size.

7. Opportunity cost inclusion and exclusion logic

STEPS distinguishes between direct financial costs and total economic costs. Indirect costs include items that are necessary for programme operation but not directly attributable to delivery, including opportunity costs. Users can include or exclude these to present results from a narrower budget perspective or a broader economic perspective.

Total economic cost = Direct cost × (1 + Indirect to direct ratio)

Explanation: The indirect to direct ratio is computed from the costing dataset for each programme type. When opportunity cost inclusion is switched off, the tool reports direct cost only.

8. Willingness to pay calculations

WTP is computed from the preference model and is interpreted as the monetary value stakeholders place on a full programme package, expressed as a compensating variation in money. In STEPS, WTP is reported per trainee per month and scaled to the cohort and scenario horizon.

WTP per trainee per month = - (ΔV non cost) / βc

Explanation: ΔV non cost is the change in systematic utility from moving from opt out to the selected programme package excluding the cost term. βc is the marginal utility of cost (negative). Dividing utility by the cost coefficient converts utility units into money units.

Total WTP (scenario) = WTP per trainee per month × Duration months × Cohort size × Number of cohorts

Explanation: Total WTP scales linearly with the number of trainees trained and the duration of training because the cost attribute is expressed per month.

Interpretation rule: WTP is a welfare measure derived from stated preferences. It is not a budget line item. In STEPS it is used as one definition of programme benefit and can be combined with epidemiological benefits in the benefit cost analysis.

9. Epidemiological benefit construction

Epidemiological benefits in STEPS represent economic cost savings from outbreaks averted or substantially reduced through improved detection and response capacity associated with the scaled training programme. The tool expresses outbreak savings as a monetary value per outbreak and multiplies this by an expected number of outbreaks averted over the planning horizon, with discounting.

Annual outbreak savings = Expected outbreaks averted per year × Value per outbreak

Explanation: Users select or set a monetary value per outbreak (Section 10). The expected outbreaks averted per year is handled through scenario assumptions in the Settings tab and is applied consistently across scenarios.

PV of outbreak savings = Σt=1..T [ Annual outbreak savings / (1 + r)t ]

Explanation: The tool discounts future outbreak savings at a user selected discount rate r over a planning horizon of T years.

Why ₹20.0 billion baseline, and why ₹8.3 billion and ₹166 billion for sensitivity

₹20.0 billion is used as a conservative central value for a large multi state outbreak where economic losses extend beyond direct health costs to include disruption to services, productivity, and public sector response costs. ₹8.3 billion (US$0.1 billion) represents a lower bound consistent with sizeable outbreaks whose economic costs remain in the single digit billions of rupees. ₹166 billion (US$2.0 billion) represents rare but plausible large scale events where macroeconomic disruption is material, consistent with high end comparators in the literature. The sensitivity tab applies these alternative values to the same expected outbreak avoidance assumptions to show how results change when outbreak severity assumptions change.

For users who want to stress test more localised events, the dropdown includes smaller values (for example ₹40 million, ₹80 million). These do not replace the baseline; they provide a conservative sensitivity check.

11. Benefit cost ratios and net present value

STEPS reports both net benefit and benefit cost ratio. The tool supports alternative benefit definitions: WTP only, outbreak savings only, or combined. Discounting is applied to benefits that accrue over the planning horizon (outbreak savings), while programme costs and WTP occur during the training period and are treated as near term.

PV(Benefits) = PV(WTP) + PV(Outbreak savings)

Explanation: When combined benefits are selected, the tool adds the total WTP (welfare measure) to the present value of outbreak savings (economic cost savings). Users can also view these separately.

NPV = PV(Benefits) - PV(Costs)
BCR = PV(Benefits) / PV(Costs)

Explanation: Net present value reports the net gain after subtracting costs. Benefit cost ratio reports benefit per rupee of cost. A ratio above 1 implies benefits exceed costs under the selected benefit definition and assumptions.

12. National simulation scaling

National scaling in STEPS is implemented by multiplying cohort level quantities by the number of cohorts delivered and by the number of implementing institutions where relevant. The scaling logic is deliberately linear to preserve transparency and to avoid hidden assumptions.

Total trainees trained = Trainees per cohort × Number of cohorts

Explanation: Scale up is represented by the number of cohorts and cohort size. If multiple institutions are represented explicitly, the tool applies the same formula within each institution and then sums totals.

Total national cost = Cost per cohort × Number of cohorts
Total national WTP = WTP per cohort × Number of cohorts

Explanation: Costs and WTP scale with the number of trainees trained. Outbreak savings can also be scaled by programme size if the Settings tab links expected outbreak avoidance to the scale of trained graduates.

13. Sensitivity analysis framework

The Sensitivity tab recomputes results under alternative assumptions to support robust decision making. The key sensitivity dimensions are as follows.

Sensitivity dimensions and what changes in the calculations
Sensitivity dimension What the user changes What recalculates
Benefit definition options Select WTP only, outbreak savings only, or combined. PV(Benefits), NPV, and BCR update based on the selected benefit components.
Endorsement adjusted WTP Apply an endorsement or acceptability adjustment factor between 0 and 1 to WTP. PV(WTP) is multiplied by the factor before combining with outbreak savings.
Outbreak value sensitivity Select outbreak value, including baseline ₹20.0 billion and sensitivity values ₹8.3 billion and ₹166 billion, plus smaller conservative values. PV(outbreak savings) scales linearly with the selected value per outbreak.
Planning horizon and discounting Change T (years) and discount rate r. PV(outbreak savings), NPV, and BCR update using the discounting equation.
Interpretation of sensitivity tables Compare base case to alternatives. The tool highlights which assumptions drive conclusions and where rankings are stable or unstable.

Table note: Sensitivity results are not alternative facts. They are structured counterfactuals that show how conclusions move under plausible and deliberately conservative settings.

15. Copilot interpretation and JSON export

The Copilot tab generates a structured output designed for policy brief drafting. It provides two elements: a short narrative text describing the scenario and results, and a JSON payload that can be pasted into Microsoft Copilot to request a formatted brief.

What the Copilot tab generates

The generated content includes the scenario configuration (programme type, mentoring, delivery mode, career incentive, duration, cost), scale inputs (cohort size and number of cohorts), global assumptions (planning horizon, discount rate, outbreak value, benefit definition, opportunity cost inclusion), and the headline outputs (costs, WTP, outbreak savings, net benefit, BCR).

What Copilot does not do

Copilot does not recalculate the model, validate sources, or change assumptions. It drafts text based on the JSON you provide. Users should treat Copilot output as a drafting assistant and should always verify that the narrative matches the Results and Sensitivity tabs.

16. Summary of assumptions and limitations

Assumptions and limitations to report with STEPS outputs
Topic Assumption used by STEPS Implication for interpretation
Preference evidence and WTP WTP is derived from a mixed logit model estimated on stated choices over defined attribute ranges. WTP is a welfare measure, not a cash flow, and is most credible within the attribute ranges used in the study.
Cost slider range The cost per trainee per month input is constrained to the preference study range. Reduces extrapolation risk and supports credible welfare comparisons.
Response capacity Fixed at within 7 days and treated as an outcome rather than a design input. Avoids double counting and ensures a clean separation between inputs and benefits.
Outbreak savings values Baseline ₹20.0 billion per large outbreak with sensitivity values ₹8.3 billion and ₹166 billion, plus smaller conservative dropdown values. Outbreak benefit results are sensitive to outbreak severity assumptions. Sensitivity results should be reported alongside base case.
Discounting Discount rate and planning horizon set by the user in Settings. Long horizon benefits are sensitive to r and T; reporting should include the chosen values.
Cost breakdown shares Component shares are derived from the provided WHO, NIE, and NCDC costing data. Breakdowns are reporting aids and should be interpreted as proportional decompositions of scenario totals.
Uncertainty Preference uncertainty and outbreak uncertainty are explored through sensitivity scenarios rather than a full probabilistic model in the interface. Decisions should emphasise results that remain stable across reasonable sensitivity ranges.
